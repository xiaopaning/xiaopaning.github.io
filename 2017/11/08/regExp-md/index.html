2.RegExp对象

字面量
var str = 'he is a good boy, she is a good gril'
var reg = /\bis\b/g;
console.log(str.replace(reg, 'IS'))
构造函数
var str = 'he is a good boy, she is a good gril'
var reg = new RegExp('\\bis\\b', 'g')
console.log(str.replace(reg, 'IS'))
修饰符
  • g: gobal

  • i: 大小写

    var str = 'he is a good boy, she Is a good gril'
var reg = /\bis\b/gi;
console.log(str.replace(reg, '0'))
  • m: 多行搜索

4.字符类

一般情况下正则表达式的一个字符对应字符串中的一个字符

表达式ab\t的含义是 字母：ab + 水平制表符：’tab’

字符类
  • 我们可以使用一个元字符[]来构建一个简单的类
  • 所谓的类就是指符合某些特征的对象，一个泛指，而不是特指某个字符

  • 如表达式[abc]把字符a或者b或者c归为一类，表达式可以匹配这类的字符。

    var str = 'a1b2c3'
var reg = /[abc]/g
console.log(str.replace(reg, '0'))
字符类取反
  • 使用元字符 ^ 创建反向类／负向类
  • 反向类意思是不属于某类的内容

  • [^abc]意思是不是a或b或c的内容

    var str = 'a1b2c3'
var reg = /[^abc]/g
console.log(str.replace(reg, '0'))

5.范围类

[a-z] [a-zA-Z] [0-9]

 var str = '2017-11-15'
var reg = /[0-9-]/g;
var newStr = str.replace(reg, '.')
console.log(newStr)

注意⚠️上述的 - 是在范围内加的 -

字符 等价 意义
. [^\r\n] 出了回车换行之外的所有字符
\d [0-9] 数字字符
\D [^0-9] 非数字字符
\s [\t\n\xOB\f\r] 空白符
\S [^\t\n\xOB\f\r] 非空白符
\w [a-zA-Z_0-9] 单词字符包括字母数字下滑线
\W [^a-zA-Z_0-9] 非单词字符
预定义类

匹配 ab+数字+.
ab\d.

边界
字符 含义
^ 以xx开始
$ 以xx结束
\b 单词边界
\B 非单词边界

^ 在[]里边是非，在之外是以xx开始

var str = '@12@34'
var reg = /^@/g;
var newStr = str.replace(reg, 'Q')
console.log(newStr)

7.量词

字符 含义
? 出现零次或者一次
* 出现零次或者多次
+ 最少出现一次
{n} 出现n次
{n, m} 出现n到m次
{n,} 最少出现n次

出现20次的数字 \d{20}

出现最多10次 \d{0, 10}

\d{20}\w\d?\w+\d*\d{3}\w{3,5}\d{3,}

含义：20次数字
一次单词
零次或一次数字
最少一次单词
零次或者多次数字
3次数字
3到5次单词
最少3次数字

8.JS正则贪婪模式与非贪婪模式

贪婪模式

尽可能多的匹配

var str = '12345678';
var reg = /\d{3,6}/g;
var newStr = str.replace(reg, 'Q')
//Q78

非贪婪模式

让正则表达式尽可能少的匹配，也就是说一旦匹配成功不在继续尝试。

做法是在量词后加一个?

var str = '12345678';
var reg = /\d{3,6}?/g;
var newStr = str.replace(reg, 'Q')
//QQ78

9.分组

()

(beyond){3}

var str = 'a1b2c3d4e5'
var newStr = str.replace(/([a-z]\d){3}/g, 'Q')
//Qd4e5

或 |

var str = 'btronCasper';
var newStr = str.replace(/(btron|Casper)/g, 'Q')
//QQ

方向引用

var str = '2017-11-16'
var reg = /(\d{4})-(\d{2})-(\d{2})/g;
var newStr = str.replace(reg,'$1/$2/$3')
//2017/11/15

10.前瞻

  • 正则表达式从文本头部向尾部开始解析，文本尾部方向，称为‘前’
  • 前瞻就是在正则表达式匹配到规则的时候，向前检查是否符合断言，后顾／后瞻方向相反
  • js不支持后瞻

  • 符合和不符合特定断言称为肯定／正向匹配和否定／负向匹配

    名称 | 正则
    —|—
    正向前瞻 | exp(?=assert)
    负向前瞻 | exp(?!assert)

例子：

var str = 'a2*3'
var reg = /\w(?=\d)/g;
var newStr = str.replace(reg,'@')
// @2*3  并不匹配字母后的数字

12.test和exec方法

RegExp.prototype.test(str)

用于测试字符串参数中是否存在匹配正则表达式模式的字符串

var reg = /\w/g
reg.test('a')
// true 
//在执行一次, 结果为false
//继续执行，结果为true
//...

这个是因为lastIndex的原因

RegExp.prototype.exec(str)

  • 使用正则表达式模式对字符串执行搜索，并将更新全局RegExp对象的属性以反应匹配结果
  • 如果没有匹配的文本则返回null，否则返回一个结果数组
    • index 声明匹配文本的第一个字符的位置
    • input 存放被检索的字符串string

非全局调用

  • 调用非全局的RegExp对象的exec()时，返回数组
  • 第一个元素是与正则表达式相匹配的结果
  • 第二个元素是与RegExpObject的第一个子表达式相匹配的文本（如果有的话）
  • 第三个元素是与RegExpObject的第二个子表达式相匹配的文本（如果有的话），以此类推
var reg3 = /\d(\w)\d/
var reg4 =  /\d(\w)\d/g
var ts = '1a2b3c4d5e6'
var ret = reg3.exec(ts);

console.log(reg3.lastIndex+ '\t' + ret.index + '\t' + ret.toString())
console.log(reg3.lastIndex+ '\t' + ret.index + '\t' + ret.toString())
// 0     0    1a2,a
// 0    0    1a2,a

注意⚠️在非全局下lastIndex根本不生效

var reg3 = /\d(\w)\d/
var reg4 =  /\d(\w)\d/g
var ts = '1a2b3c4d5e6'
while(ret = reg4.exec(ts)){
  console.log(reg4.lastIndex+ '\t' + ret.index + '\t' + ret.toString())
}
//3        0    1a2,a
//7        4    3c4,c
//11    8    5e6,e

13.字符串对象方法

str.prototype.search(reg)

  • 检索字符串中指定的子字符串，或者检索与正则表达式相匹配的子字符串
  • 返回第一个匹配结果index，查找不到返回-1
  • 不执行全局匹配，忽略g，并且总是从字符串的开始进行检索

str.prototype.match(reg)

  • 检索字符串，以找到一个或者多个与regexp匹配的文本
  • 正则表达式是否有g影响结果还是蛮大的
  1. 没有g标志
  • 只能在字符串中匹配一次，如果没有找到任何结果，返回null，否则返回一个数组
  • 返回数组当中，第一个元素返回匹配的文本，而其余元素存放的是与正则表达式的子表达式匹配的文本
  • 除了常规的数组之外，返回的数组还有2个对象属性
    • index 匹配的文本字符串的起始位置
    • input 声明对 stringObject 的引用
var reg3 = /\d(\w)\d/
var ts = '1a2b3c4d5e6'
var ret = ts.match(reg3)
console.log(ret)
//["1a2", "a", index: 0, input: "1a2b3c4d5e6"]
  1. 有g标志
  • 找到字符串中所有的匹配结果
  • 没有找到返回null
  • 找到返回一个数组
  • 没有index以及input属性
var reg4 =  /\d(\w)\d/g
var ts = '1a2b3c4d5e6'
var ret = ts.match(reg4)
console.log(ret)
//["1a2", "3c4", "5e6"]
