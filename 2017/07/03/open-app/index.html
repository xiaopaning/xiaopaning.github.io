open_app

Veröffentlicht am 2017-07-03

感叹啊，背锅这么久，最终发现不是自己的问题。

之前公司要做一个点击按钮跳转app，没有下载native app，打开store，下载页面。
思想很简单，就是先让他的location.href =’my app’s domain’，去唤醒app，如果没有安装app，使用定时器，在3s后跳转到store对应下载页面。但是在iphone里，如果安装了app，点击下载按钮，会弹出一个对话框，询问是否进入app，如果我延迟3s后，才想着去点击是或者否，此时，页面会跳转到store。bug！bug！bug！

对于这种bug我们有其实有办法解决，但是并不由前端来做，而是客户端，利用 Universal Link 调起 App。

Universal Link 是什么？
简单地说就是通过一个 https 的链接，完成从浏览器直接跳转到 App 或者在没安装 App 的情况下跳转到指定页面。没有是否要进入 App 的弹窗，没有微信客户端限制跳转。

好了，这个https链接，就不关我们前端的事情了，但是为了推锅，还是稍微大概了解下这个东西。链接如下

http://www.cocoachina.com/ios/20150902/13321.html

在我们前端只要做好，调用就行了。

if (/iPhone OS 9|iPhone OS 10/i.test(ua)) {
window.location.href = '协议';
return;
}

当浏览器切换到后台后，我们需要清除代码中的延迟下载，以防当我们再一次返回页面后，页面跳转到 App 下载地址。

document.addEventListener("visibilitychange", callAppSuccess);
document.addEventListener("webkitvisibilitychange", callAppSuccess);
document.addEventListener("msvisibilitychange", callAppSuccess);
document.addEventListener("blur", callAppSuccess);

// delayDownload ：如果本地app不存在，需要为下载app时，所定义的延时器

function callAppSuccess () {
    if(delayDownload) {
        clearTimeout(delayDownload);
     }
}

上面的方法是ios9以及10的，对于安卓还是普遍使用iframe去做。

步骤1:首先判断浏览器的类型

var ua = navigator.userAgent;

步骤2:iframe 调起 App

if ($('#open_app').size()) {
    $('#open_app').remove();
}
var naUrl = 'app's domain';
var iframeHtml = '<iframe src="' + naUrl + '" id="open_app" width="0" height="0" style="border: 0;display: none;"/>';
$(document.body).append(iframeHtml);

步骤3:判断本地是否安装了app

若 iframe 没有成功调起 App，则表示本地没有安装 App，那么此时页面跳转到 App 的下载页

 var startTime = new Date().getTime();
 var delayDownload = setTimeout(function() {
    if (new Date().getTime() - startTime < 2000) {
        var isIos = (/iphone|ipad|ipod/i).test(ua);
        var url = isIos? 'app isIos的下载链接' : '安卓的下载地址';
        window.location.href = url;
    }
}, 1000);
