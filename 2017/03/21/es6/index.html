ES6 基础

Posted on 2017-03-21

let和const命令

1. let命令

let命令，用来声明变量，用法类似于var，但是只在let命令所在的代码块内有效。
如for语句

for (let i = 0; i < 10; i++) {}
console.log(i);

上列代码，计数器i只在for循环体内有效，在循环体外引用就会报错。

在看下列两个例子：

var a = [];
for (var i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6]();

结果是 10

var a = [];
for (let i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6]();

结果是 6
注意：

$ 不存在变量提升
不允许重复声明

console.log(bar); // 报错ReferenceError
let bar = 2;

特殊情况：

var tmp = 123;
if (true) {
  tmp = 'abc'; // ReferenceError
  let tmp;
}

由于if块内，重新声明了tmp, 导致报错，在语法上称之为“暂时性死区”

$ 不允许重复声明

2. 块级作用域

ES5 只有全局作用域和函数作用域，没有块级作用域，导致块内变量泄漏到全局。

有了块级作用域，外层作用域将无法读取内层作用域的变量

{
    let insane = 'Hello World'
};

如上例，let实际为 JavaScript 新增了块级作用域

3. const命令

const声明一个只读的常量。一旦声明，常量的值就不能改变。

const PI = 3.1415;

const来说：

改变常量的值会报错
只声明不赋值，就会报错
声明所在的块级作用域内有效
同样存在暂时性死区，只能在声明的位置后面使用
不可重复声明

变量的解构赋值

1. 数组的解构赋值

模式匹配

let [a, b, c] = [1, 2, 3];

嵌套式

let [foo, [[bar], baz]] = [1, [[2], 3]];

若解构不成功，变量的值就等于 undefined

另一种情况是不完全解构，即等号左边的模式，只匹配一部分的等号右边的数组

let [x, y] = [1, 2, 3]; // x = 1, y = 2
let [a, [b], d] = [1, [2, 3], 4]; // a = 1, b = 2, c = 4

如果等号的右边不是数组， 将会报错

let [foo] = 1;
let [foo] = false;
let [foo] = NaN;
let [foo] = undefined;
let [foo] = null;
let [foo] = {};
//报错

再来下面的例子：

let [x, y = 'b'] = ['a']; // x='a', y='b'
let [x, y = 'b'] = ['a', undefined]; // x='a', y='b'

ES6 内部使用严格相等运算符（===），判断一个位置是否有值。所以，如果一个数组成员不严格等于undefined，默认值是不会生效的。
例子：

let [x = 1] = [undefined];   x // 1
let [x = 1] = [null];       x // null

2. 对象的解构赋值

let { foo, bar } = { foo: "aaa", bar: "bbb" };

注意

let { foo: baz } = { foo: "aaa", bar: "bbb" };

上例中，foo是匹配的模式，baz才是变量。真正被赋值的是变量baz，而不是模式foo。

和数组一样，解构也可以用于嵌套结构的对象

let obj = {
  a: [
    'Hello',
    { b: 'World' }
  ]
};

let { p: [a, { b }] } = obj;

默认值生效的条件是，对象的属性值严格等于undefined。

var {x = 3} = {x: undefined};        x // 3

var {x = 3} = {x: null};        x // null
